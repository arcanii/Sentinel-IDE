What's happening

snc compiled crypto.sentinel, then looked for libsentinel_runtime.a in target\release\ to link against. This environment has sentinel_runtime.lib (the MSVC staticlib) instead, so the link step can't complete.

Best-known steps ⎘ Copy

# confirm what snc is linking against snc build crypto.sentinel --verbose # check the archive snc expects (vs the .lib that's present) dir %SENTINEL_HOME%\target\release\
